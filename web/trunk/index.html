CLERK ignition
Storage & Application Engine

 

The Clerk Ignition engine is a storage and application server in one. It's Fast, flexible and free. Small and easy to manage.

- OR it will be be once it's reached a more mature state.This project is still in the Alpha-stage.

What can it do? Where would I use it?

  • Building dynamic webapplications - without using a SQL-database. Add a webserver.
  • Building rich-client applications (desktop-application). Add your favorite GUI-toolkit.
  • Embeded/devices.

The ignition engine has flexible interfaces to I/O and physical storage allowing it to work in many different settings.

How does it work?
The ignition engine stores data in treelike structures alot like XML. The structure can include functions and expressions written in the programming language Ignite (see below).

Using this server is a blend between using af distributed spreadsheet, a relational database and XML.

Ignite Programming Language
Ignite looks somewhat like C/PHP/Java on firste sight - but it works a bit different. First of all; you are building trees in Ignite. Trees and lists. Next the language has aspects of functional style languages.

It is a far cousin of JavaScript in many respects. It has dynamic typing, some prototyping...

Well... more details will follow as the project matures.

Present state
The first "Hello world" program has run on the engine, but alot of work remains before the enigine is ready for production use.
